# 00 · Tech Stack

Foundational, **shared documentation** of the tools, languages, frameworks,
runtimes and cross-cutting conventions the project is built on. This is the
common ground every agent reads before working — the base context that the
other KB layers assume.

Put here things like:

- Languages, frameworks and runtime versions.
- Repository layout and build/run commands.
- Naming, formatting and code conventions.
- Shared primitives and contracts the system composes on.
- Environment and tooling setup.

Not owned by a single skill — it is base context for all of them.
