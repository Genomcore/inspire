# 03 · Prototypes

Prototypes exist to **create clarity and knowledge**, not production code.
Building a thing is often the fastest way to learn what it should be; what the
prototype teaches flows back into features ([`02_features`](../02_features)),
specs ([`04_specs`](../04_specs)) and UI ([`05_ui`](../05_ui)).

- **Skill:** `inspire-prototype`.
- **Two shapes:**
  - [`horizontal/`](horizontal) — **breadth**. A single, shallow prototype that
    walks across the whole product so the shape of the system becomes legible.
  - [`verticals/`](verticals) — **depth**. Several focused prototypes, one per
    subfolder (`verticals/{name}/`), each taking a single flow end-to-end.

Each prototype should end with a short note on **what it clarified** — the
knowledge is the deliverable.
