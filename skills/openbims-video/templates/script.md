## Voice & caption rules (read before authoring)

**Mock data is visual support, not narrative substance.** The camera lands on a hero entity (a row, a card, a metric) so the viewer has something concrete to look at. The narration explains the **product feature** at PDD level — what the section does, what it composes, what capabilities it gives. Never recite the mock numbers, IDs, or names; they're invented demo data and citing them turns a product tour into a fake case study.

**Anti-patterns**:

```
❌ "Maria.garcia read 47 records — actor IP, trace, hash chain — one immutable line."
✅ "Every audit event carries actor, resource, trace ID — drill into any row for full
    cross-module correlation."

❌ "v47, 14 days uptime, 89 requests a second — same scope as Auth's patient pool."
✅ "Each portal is versioned with continuous deploys, exposing uptime, request rate
    and error-rate metrics out of the box."

❌ "rule-007 Data Access Spike fires on 200% baseline."
✅ "Rules are queries over the same logs and metrics — versioned, scoped, evaluated
    on a fixed cadence."

❌ "Workflows run on the native Restate engine."
❌ "Console users live on Ory Kratos under the system_user schema."
❌ "Functions are V8 isolates; OpenTelemetry feeds the dashboards."
✅ "Workflows are durable orchestration with exactly-once steps and replay."
✅ "Console users live under one identity schema, scope openbims."
✅ "Functions run as sandboxed isolates fired at hook points."
```

**No internal implementation tech in the voice.** Implementation details that may change without changing the user-facing product MUST stay out of caption / narration: `Ory Kratos / Keto / Hydra`, `Restate`, `V8 isolates`, `OpenTelemetry`, `AES-256-GCM`, `ClickHouse`, internal SDK names. The product story has to survive a backend swap. **DO** name public-facing artifacts the user actually picks: external connectors (`Kubernetes`, `Docker`, `Slurm`, `AWS`, `GCP`, `HashiCorp Vault`, `Airflow`, `Nextflow`, `n8n`), open standards (`OAuth2`, `FHIR`, `GDPR Art. 30`, `ISO 27001`, `21 CFR Part 11`), file formats, and product concepts (`ReBAC`, `durable workflows`, `hash-chained ledger`, `sandboxed isolates`).

**Rules**:

- Captions: feature-level labels (`Catalog`, `Detail`, `Versioned`, `Hash Chain`). Never entity IDs (`evt-005`, `agt-variant-interp`).
- Narration: cover the *feature*, in PDD vocabulary. Mock IDs / metrics / personas stay off-mic.
- Hero entities still belong in `**Proof**` so `mockplay` knows where to point the camera; they just don't end up in the voice or chyron.
